TESTING STRATEGY
------------------------------------------------------------

When new data is added to one of the data sources:
  1. Re-run: python etl/ingest_to_snowflake.py
     → RAW tables refreshed in Snowflake

  2. Re-run: python etl/spark_transform.py
     → CURATED.BOOKINGS and CURATED.FEEDBACK updated via Spark

  3. Re-run: snowsql -f sql/snowflake/03_bo1_premium_revenue.sql
             snowsql -f sql/snowflake/04_bo2_satisfaction.sql
     → ANALYTICS layer and JSON_EXPORT table updated

  4. Dashboard reflects updated metrics automatically.

  5. Run: pytest tests/ -v
     → 37 assertions validate data quality and pipeline logic
